Command Overlay Explorations

5 fundamentally different interaction models for a zero-friction command overlay. Each appears as soon as you start typing — no modifier key, no leading character. Goal: non-technical users type at expert speed via natural language + aggressive completion.

VARIATION 01
Spotlight
Classic centered search bar with fuzzy-matched results list. Categories, descriptions, and chord shortcuts shown inline. The baseline — familiar, proven, minimal learning curve.
fuzzy search result list baseline
VARIATION 02
Ghost Text
Thin bottom strip with Copilot-style inline completion. Gray ghost text predicts your intent; Tab to accept. Feels like the computer is reading your mind.
inline prediction Tab to accept minimal UI
VARIATION 03
Sentence Chips
Structured slot-filling with pill-shaped chips. Build a sentence: [Verb] + [Object] + [Destination]. Tab advances slots; number keys pick options. Tactile and guided.
slot filling guided structured
VARIATION 04
Predictive Grid
After each word, a 3×3 grid of likely next words appears. Press 1-9 to pick. "Move files to Downloads" in 3 keystrokes. Feels like a game — fast, rewarding, zero typing required.
number keys zero typing game-like
VARIATION 05
Cascading Columns
Miller-column metaphor applied to commands. Same j/k/h/l navigation you already know. If you can browse files, you can browse commands.
miller columns familiar nav progressive

Design Brief

These prototypes explore how a file manager can support both power users (vim chords) and non-technical users (natural language) through a single overlay that appears on first keystroke. Each variation is self-contained HTML — open in any browser. They share a common aesthetic (dark theme, electric violet accent) but differ fundamentally in interaction model.
